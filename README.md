# NetDevOps入门

Git使用指导
==============
1. echo "# NetDevOps" >> README.md
字面意思：把 # NetDevOps 这行文字，写入到一个名叫 README.md 的文件中。
大白话解释：这是在写项目说明书。它会在你的当前文件夹里创建一个文件，里面写着一行大标题叫 "NetDevOps"。就像你买了个新本子，在第一页写上了名字。
2. git init
字面意思：初始化（Initialize）Git 仓库。
大白话解释：告诉你的电脑：“从现在开始，请在这个文件夹里开启 Git 监控系统”。敲完这行命令后，文件夹里会多出一个隐藏的 .git 文件夹，此时本地仓库就建好了。
3. git add README.md
字面意思：把 README.md 文件添加到暂存区。
大白话解释：就像是在淘宝购物时**“把商品加入购物车”**。告诉 Git：“这个文件我改好了，准备一会儿打包存起来”。如果你以后写了 10 个 Python 脚本想一起上传，通常会用 git add .（把当前目录下所有改动都加进购物车）。
4. git commit -m "first commit"
字面意思：提交（commit）改动，并附带留言（-m / message）"第一次提交"。
大白话解释：这就是**“游戏存档”**！把刚才购物车里的东西正式打包成一个包裹。-m 后面必须跟着引号，里面写你这次改了什么东西的备忘录。以后无论代码写得多乱，你都能随时读档回到这个叫做 "first commit" 的状态。
5. git branch -M main
字面意思：把当前的分支重命名为 main。
大白话解释：Git 默认的主线原本叫 master（主人），但因为近几年的文化原因，GitHub 提倡把主干分支统称为 main（主要的）。这行命令就是顺应时代，把你的主分支名字改成 main。
6. git remote add origin https://github.com/Altair12138/NetDevOps.git
字面意思：添加一个远程（remote）仓库，给它起个别名叫 origin，地址是你的 GitHub 网址。
大白话解释：这步是在**“牵红线”**！本地仓库在你的电脑上，云端仓库在 GitHub 上，它们原本互不相识。这行命令告诉本地电脑：“以后你要上传代码，就发到这个 Altair12138 的网址里去”，并且给这个长长的网址起个短一点的外号，叫 origin（起源/远端）。
7. git push -u origin main
字面意思：把本地的 main 分支代码，推送（push）到 origin 远端的 main 分支。
大白话解释：最后一步，发射升空！ 把你刚才打好的包裹，顺着网线上传到 GitHub 上。
-u 的意思是“记住这次路线”。只要你敲过一次这个完整的命令，以后你每天写完新代码，只需要敲简简单单的两个词 git push，电脑就会自动知道要把代码推送到 GitHub 上啦！
