## 正则表达式
正则表达式的核心思想是：**“通过描述字符的特征”来匹配文本**。

为了让你彻底明白刚才的代码，我们先看几个最常用的“基础积木”，然后再逐一拆解作业中的代码。

### 0. 基础符号速查（本次作业用到的）

| 符号 | 含义 | 记忆口诀 |
| :--- | :--- | :--- |
| `\d` | 匹配一个数字 (0-9) | **d**igit (数字) |
| `\w` | 匹配一个字母、数字或下划线 | **w**ord (单词字符) |
| `\s` | 匹配一个空白字符 (空格、Tab) | **s**pace (空白) |
| `\S` | 匹配一个**非**空白字符 (除了空格以外的任何东西) | **S** (大写表示“非”) |
| `.` | 匹配除了换行符以外的任意字符（但在 `[]` 中只表示小数点） | 通配符 |
| `+` | 表示前面的字符出现 **1次或多次** | 贪婪匹配 |
| `()` | **分组捕获**，提取括号里的内容 | “我要这个！” |
| `[]` | **字符集合**，匹配方括号内列出的任意一个字符 | “挑一个” |

### 1. 详解作业 1：解析 MAC 地址表

**目标字符串：**
`'166    54a2.74f7.0326    DYNAMIC    Gi1/0/11'`

**使用的正则：**
`r'(\d+)\s+([0-9a-f.]+)\s+(\w+)\s+(\S+)'`

**详细拆解：**

1.  **`(\d+)`** —— **提取 VLAN**
    *   `\d`：找数字。
    *   `+`：找连续的数字（1个或多个）。
    *   `()`：**括号把它们包起来**，因为这是我们要提取的数据（VLAN ID 166）。

2.  **`\s+`** —— **跳过空格**
    *   `\s`：找空格。
    *   `+`：找连续的空格。
    *   **没有括号**：因为我们只是用它来定位，不需要把空格提取出来。

3.  **`([0-9a-f.]+)`** —— **提取 MAC 地址**
    *   `[...]`：这是个自定义集合。
    *   `0-9`：允许数字。
    *   `a-f`：允许小写字母 a 到 f（十六进制）。
    *   `.`：允许点号。
    *   `+`：只要是上面这些字符组成的连续串，都算进来。
    *   `()`：提取出来（MAC地址）。

4.  **`\s+`** —— **跳过空格**

5.  **`(\w+)`** —— **提取类型 (DYNAMIC)**
    *   `\w`：找单词字符（字母/数字）。
    *   `+`：连续的。
    *   `()`：提取出来。

6.  **`\s+`** —— **跳过空格**

7.  **`(\S+)`** —— **提取接口 (Gi1/0/11)**
    *   这里为什么要用 `\S` 而不是 `\w`？
    *   因为接口名称里包含了 `/` (斜杠)。`\w` 包含字母数字，但不包含 `/`。
    *   `\S` 的意思是“**只要不是空格，我全都要**”。它会一直匹配到字符串结束或者遇到下一个空格为止。

### 2. 详解作业 2：解析 ASA 防火墙连接表

**目标字符串：**
`'TCP server 172.16.1.101:443 localserver 172.16.66.1:53710...'`

**使用的正则：**
`r'^(\w+)\s+server\s+([\d.]+):(\d+)\s+localserver\s+([\d.]+):(\d+)'`

**详细拆解：**

1.  **`^(\w+)`** —— **提取协议 (TCP)**
    *   `^`：表示**必须从字符串的开头**开始匹配。
    *   `(\w+)`：提取第一个单词（TCP）。

2.  **`\s+server\s+`** —— **定位锚点**
    *   这是一段**固定文本**。
    *   我们告诉正则：“找到协议后，后面必须紧跟着空格，然后是单词 `server`，再是空格”。
    *   这部分不需要提取，所以不加括号，它只是用来确保我们找对了位置。

3.  **`([\d.]+)`** —— **提取 Server IP**
    *   `[\d.]`：集合里包含数字 `\d` 和点号 `.`。
    *   `+`：匹配连续的 IP 地址字符。
    *   `()`：提取出来。

4.  **`:`** —— **匹配冒号**
    *   IP 和端口中间有个冒号，直接写 `:` 就能匹配它自己。

5.  **`(\d+)`** —— **提取 Server Port**
    *   `\d+`：匹配连续数字（端口号）。
    *   `()`：提取出来。

6.  **`\s+localserver\s+`** —— **定位锚点**
    *   和前面的 `server` 一样，这是用来分隔两组 IP 的关键词。找到这个词，说明后面跟着的就是 Client IP 了。

7.  **`([\d.]+)`** 和 **`(\d+)`** —— **提取 Client IP 和 Port**
    *   原理同上。

### 总结一下核心技巧

1.  **找数据 vs 找分隔符**：
    写正则时，要把字符串看作“**我要的数据**”和“**我不关心的分隔符**”的组合。
    *   用 `()` 包裹你要的数据。
    *   用 `\s+`、`:`、`server` 等描述分隔符，帮助正则定位。

2.  **善用 `\S` (非空白)**：
    当你不知道数据里有没有特殊符号（比如 MAC 地址里的点、接口里的斜杠、邮箱里的 @），但你知道它是连续的一串字符且中间没空格时，用 `(\S+)` 是最省事的写法。

3.  **特殊符号要转义**：
    虽然今天的作业没用到，但如果你的文本里本身就有 `(`、`[`、`+` 这些符号，你需要写成 `\(`, `\[`, `\+` 来匹配它们原本的样子。
